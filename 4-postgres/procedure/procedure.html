Les procédures stockées dans PostgreSQL sont un moyen puissant d'encapsuler des blocs de logique de programmation SQL ou PL/pgSQL pour une réutilisation et une gestion centralisée. Voici une vue d'ensemble détaillée sur les procédures stockées dans PostgreSQL, couvrant la création, l'utilisation et les meilleures pratiques.

1. Création de Procédures Stockées

a. Utilisation de PL/pgSQL

PL/pgSQL (Procedural Language/PostgreSQL) est le langage procédural intégré de PostgreSQL, utilisé pour écrire des fonctions et des procédures stockées.

Exemple de création d'une procédure stockée

CREATE OR REPLACE FUNCTION add_numbers(a INTEGER, b INTEGER)
RETURNS INTEGER AS $$
BEGIN
    RETURN a + b;
END;
$$ LANGUAGE plpgsql;

b. Utilisation d'autres langages

PostgreSQL supporte d'autres langages comme PL/Python, PL/Perl, PL/Tcl, etc. Exemple avec PL/Python :

CREATE OR REPLACE FUNCTION add_numbers(a INTEGER, b INTEGER)
RETURNS INTEGER AS $$
    return a + b
$$ LANGUAGE plpythonu;

2. Exécution de Procédures Stockées

Appel d'une fonction

Pour appeler la fonction add_numbers créée ci-dessus :

SELECT add_numbers(2, 3);

3. Gestion des Transactions

Les procédures stockées peuvent gérer des transactions, des points de sauvegarde et des annulations.

Exemple avec gestion de transaction

CREATE OR REPLACE FUNCTION transfer_funds(sender INTEGER, receiver INTEGER, amount NUMERIC)
RETURNS BOOLEAN AS $$
BEGIN
    -- Début de la transaction
    BEGIN;

    -- Déduire le montant du compte de l'expéditeur
    UPDATE accounts SET balance = balance - amount WHERE account_id = sender;

    -- Ajouter le montant au compte du destinataire
    UPDATE accounts SET balance = balance + amount WHERE account_id = receiver;

    -- Vérifier si les mises à jour sont réussies
    IF NOT FOUND THEN
        -- Si une mise à jour échoue, annuler la transaction
        ROLLBACK;
        RETURN FALSE;
    END IF;

    -- Confirmer la transaction
    COMMIT;
    RETURN TRUE;

EXCEPTION WHEN OTHERS THEN
    -- En cas d'erreur, annuler la transaction
    ROLLBACK;
    RETURN FALSE;
END;
$$ LANGUAGE plpgsql;

4. Procédures et Fonctions

Il est important de distinguer les procédures (introduites dans PostgreSQL 11) des fonctions :

  • Fonctions : Utilisées pour calculer et retourner une valeur. Elles peuvent être appelées dans les requêtes SQL.
  • Procédures : Utilisées pour exécuter une séquence d'instructions mais ne peuvent pas retourner de valeur directement. Elles peuvent gérer des transactions (COMMIT et ROLLBACK).

Création d'une procédure

CREATE PROCEDURE transfer_funds(sender INTEGER, receiver INTEGER, amount NUMERIC)
LANGUAGE plpgsql
AS $$
BEGIN
    -- Début de la transaction
    BEGIN;

    -- Déduire le montant du compte de l'expéditeur
    UPDATE accounts SET balance = balance - amount WHERE account_id = sender;

    -- Ajouter le montant au compte du destinataire
    UPDATE accounts SET balance = balance + amount WHERE account_id = receiver;

    -- Vérifier si les mises à jour sont réussies
    IF NOT FOUND THEN
        -- Si une mise à jour échoue, annuler la transaction
        ROLLBACK;
    ELSE
        -- Confirmer la transaction
        COMMIT;
    END IF;

EXCEPTION WHEN OTHERS THEN
    -- En cas d'erreur, annuler la transaction
    ROLLBACK;
END;
$$;

Appel d'une procédure

CALL transfer_funds(1, 2, 100.0);

5. Paramètres et Types de Retour

Paramètres IN, OUT et INOUT

  • IN : Paramètre d'entrée (par défaut).
  • OUT : Paramètre de sortie.
  • INOUT : Paramètre qui sert à la fois d'entrée et de sortie.

Exemple avec OUT et INOUT

CREATE OR REPLACE FUNCTION calculate_area_perimeter(length NUMERIC, width NUMERIC, OUT area NUMERIC, INOUT perimeter NUMERIC)
RETURNS RECORD AS $$
BEGIN
    area := length * width;
    perimeter := 2 * (length + width);
END;
$$ LANGUAGE plpgsql;

Appel de la fonction avec paramètres OUT et INOUT

SELECT * FROM calculate_area_perimeter(5, 3, 0);

6. Gestion des Exceptions

Utilisation du bloc EXCEPTION

CREATE OR REPLACE FUNCTION divide_numbers(a NUMERIC, b NUMERIC)
RETURNS NUMERIC AS $$
BEGIN
    RETURN a / b;
EXCEPTION
    WHEN division_by_zero THEN
        RAISE NOTICE 'Cannot divide by zero';
        RETURN NULL;
END;
$$ LANGUAGE plpgsql;

7. Opérations de Maintenance

Mise à jour des procédures

Pour mettre à jour une procédure existante, utilisez CREATE OR REPLACE :

CREATE OR REPLACE FUNCTION add_numbers(a INTEGER, b INTEGER)
RETURNS INTEGER AS $$
BEGIN
    RETURN a + b;
END;
$$ LANGUAGE plpgsql;

Suppression des procédures

DROP FUNCTION add_numbers(INTEGER, INTEGER);
